Fixed — 3 remaining bugs

Fixed — Bug 1 — Double notify in AuthService.updateProfile()
The duplicate lines are still there. Lines 476-477 of AuthService.ts set this.currentUser and call notifyListeners() twice in a row. Every settings save triggers two re-renders across the whole app. Delete the second pair of identical lines.
Fixed
Bug 2 — FCM token stored as single field, Cloud Function expects an array
PushNotificationService.saveTokenToFirestore() still writes fcmToken: token (a string) to Firestore. But your new Cloud Function reads userData.fcmTokens (an array) and calls sendEachForMulticast({ tokens: fcmTokens }). The token will be saved but never read — push notifications will silently fail on mobile. Fix: change saveTokenToFirestore to use FieldValue.arrayUnion(token) to write into an fcmTokens array, matching what the Cloud Function expects.
Fixed
Bug 3 — GoalsService still throws on unauthenticated guest users
GoalsService still throws "User not authenticated" if userId is null. This is fine for logged-in users (now wired correctly), but it means any guest user who visits the Goals screen will see a silent crash. Add a graceful empty-state return instead of a throw when this.userId is null, so guest users see an "Sign in to use Goals" prompt rather than an error.
